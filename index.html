GitHub Repository - New Create Tasks

Steps on How-To "Commit" to Front-End Learning

  • Find new location for your next assignment (i.e. correct folder on your computer).
  • Create new folder in VSC for that week's assignment.
  • Create new .html and .js files
  • Open your Terminal and find the corresponding (correct) folder to link with your new GitHub repository
  • Create new repository (of same name as new folder in VSC) in your GitHub library

    • BONUS: Copy/pasta the GitHub "Create a new repository" steps to streamline creation of new folder + repository connection (and avoid typos!).

  • Press enter (execute commands in Terminal). You should see a status-y message that shows confirmation of the new repository + folder connection being created.

  • Use "git add ." to add the "missing" .html and .js files from your local machine's (computer) folder.
  • Use "git status" to check that those were properly found and added for commitment (saving/uploading to GitHub repository). Note: Use "git status" often to check on how your new folder(s) and/or GitHub repo(s) are connecting (updating file changes).

    • Files that have not yet been added - but were successfully found - will be shown in green text.

  • Use "git commit -m "(description)" to leave a note in the GitHub repository's version history about what was added/changed/etc.
